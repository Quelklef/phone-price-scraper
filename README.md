# Phone Price Scraper

## 1. What is this?

This project scrapes phone listings for the lowest price matching an input `(seller, model, storage, condition)` query.

It is primarily AI-written, with human guidance and review.

Attributes searched:

- Sellers:
  <a href="https://swappa.com" target="_blank" rel="noopener noreferrer">swappa</a>,
  <a href="https://www.ebay.com" target="_blank" rel="noopener noreferrer">ebay</a>,
  <a href="https://www.amazon.com" target="_blank" rel="noopener noreferrer">amazon</a>,
  <a href="https://www.backmarket.com" target="_blank" rel="noopener noreferrer">backmarket</a>
- Models: any string, defaults to `Google Pixel 6a` through `Google Pixel 10 Pro Fold`
- Storages: any integer (GB), defaults to `128GB`, `256GB`, `512GB`
- Conditions: `good`, `best`. The specific meanings of these words vary by seller

## 2. How do I use it?

You can run it against sample cached HTTP data with:

```bash
nix run github:quelklef/phone-price-scraper?ref=sample-data
```

Or against live data with the default app:

```bash
nix run github:quelklef/phone-price-scraper
```

## 4. Gotchas

- Multi-variant listings are intentionally excluded for simplicity. Example: listings like `"Pixel 7 / 7 Pro 128GB"` are ignored instead of trying to infer a single exact model variant.

## 5. Verifying Scraper Correctness

Web scraping is brittle: marketplace DOM and page behavior change over time. A marketplace can drift, causing its scraper to silently return incorrect results.

This project uses two safeguards:

1. HTTP cache committed to git

   Network responses are cached in `data/` and committed to version control. This gives stable, reproducible runs by default. Pulling fresh live HTML is explicit/intentional, instead of happening silently every run.

2. Known-good price checks

   `data/known-prices.json` stores manually-verified prices for specific `(seller, model, storage condition)` quads. Then, during a usual program run, generated prices are checked against the known-good prices. Mismatches are treated as possible indications of a scraper bug

### Adding a verified price

Price verification is manual; there is no `add-known-price` command. Spin up an AI agent and work with it to:

- Choose a quad to verify (human or ai),
- Inspect live seller query pages (human),
- Record the known-good lowest price (usually ai),
- And fix seller parser logic when mismatches appear (usually ai or collab)

Verified prices are marked in program output with `(v)`.
